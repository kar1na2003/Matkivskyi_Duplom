# ModusMate: Реалізація Обробки Та Аналізу Зображень У Вбудованих Системах

## Що Це За Проєкт

ModusMate це програмно-апаратний комплекс для обробки зображень у реальному часі на edge-AI платформі Infineon PSoC Edge E84. Проєкт об'єднує:

- вбудовану прошивку для мікроконтролера,
- host-side інструменти на Python (CLI, GUI, бенчмарки, навчання, керування моделями),
- повний пакет матеріалів кваліфікаційної роботи у LaTeX.

Цей репозиторій є одночасно кодовою базою системи та репродукованим артефактом дипломної роботи.

## Короткий Підсумок Дипломної Роботи

Мета роботи полягала у створенні повного конвеєра комп'ютерного зору для embedded-платформи з обмеженими ресурсами пам'яті та обчислень:

1. захоплення кадру з USB-камери,
2. передобробка кадру класичними алгоритмами CV,
3. інференс нейронної мережі для задачі Rock-Paper-Scissors,
4. телеметрія, керування та візуалізація результатів із ПК.

У межах роботи реалізовано та систематизовано 51 алгоритм обробки зображень, виконано порівняння за точністю, латентністю, швидкодією та ресурсною вартістю, а також сформовано практичні рекомендації для edge-AI систем реального часу.

## Основні Результати

- Побудовано стабільний end-to-end тракт від камери до класифікації.
- Реалізовано бібліотеку класичних препроцесорів (градієнтні, порогові, морфологічні, текстурні та інші).
- Інтегровано змінну модель NN без переписування основної логіки firmware.
- Доведено, що градієнтні та морфологічні препроцесори часто дають найкращий практичний компроміс між якістю та вартістю обчислень у заданих умовах.
- Підготовлено формалізований звіт із таблицями, графіками, додатками та відтворюваними скриптами.

## Архітектура Системи

### Board-Side

- Платформа: PSoC Edge E84 (ARM Cortex-M55).
- Обробка кадру та інференс виконуються на firmware-пайплайні.
- Комунікація з ПК через UART-протокол із контрольними сумами, ACK і повторними спробами.

### Host-Side

- Python-пакет для:
  - керування платою,
  - запуску GUI-прев'ю,
  - запуску benchmark/sweep,
  - тренування пер-алгоритмних моделей,
  - експорту та прошивки моделей.

### Взаємодія

- Host передає команди та параметри алгоритмів.
- Board повертає телеметрію, FPS, результати детекції та кадри прев'ю.
- Модель можна замінювати через стандартний pipeline flash без зміни архітектури застосунку.

## Структура Репозиторію

- board-side/
  - прошивка ModusToolbox, задачі захоплення кадру, обробки, інференсу.
- host-side/
  - Python CLI/GUI, тести, скрипти бенчмарків, тренування, реєстр моделей.
- report/
  - LaTeX джерела дипломної,
  - дані (CSV/XLSX),
  - графіки та ілюстрації,
  - скрипти генерації фігур і таблиць,
  - зібраний PDF.

## Технологічний Стек

- Embedded: ModusToolbox, C/C++, FreeRTOS, BSP/PDL компоненти.
- Host: Python, NumPy, Matplotlib, serial-комунікація, Tk GUI.
- ML: модельний registry, обмінні ABI-інтерфейси, workflow навчання та прошивки.
- Документація: LaTeX (повний звіт із додатками).

## Експериментальна Частина

Оцінювання виконувалось у трьох практичних сценаріях подачі зображень:

1. прямий цифровий канал,
2. оптичне захоплення з екрана,
3. зйомка проєкції на стіну.

Порівнювались:

- якість розпізнавання,
- латентність та пропускна здатність,
- стабільність до оптичних спотворень,
- витрати пам'яті та обчислень.

## Швидкий Старт

### 1. Host-Оточення

```bash
cd host-side/host
python3 -m venv .venv
.venv/bin/pip install -e .[dev]
.venv/bin/pip install kagglehub
```

### 2. Збірка Firmware

```bash
cd board-side/ws-camera-imgproc-usb/camera-imgproc-usb
make build -j
```

### 3. Запуск GUI

```bash
cd host-side/host
./run_gui.sh
```

### 4. Бенчмарк

```bash
cd host-side/host
modusmate-bench --port <PORT> --algo passthrough --limit 50
```

### 5. Прошивка Моделі

```bash
cd host-side
python -m modusmate_host.models flash <MODEL_NAME> --port <PORT> --fw <ABS_FW_PATH>
```
